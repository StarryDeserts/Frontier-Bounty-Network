## Known live limitation

`claim_bounty` was intentionally not forced on testnet:
- it requires a `KillProof` object
- the current package only exposes proof constructors in `#[test_only]` code
- there is no production CLI path in the repository that naturally creates a valid live proof

## Accepted deviation during hardening

The initial indexer listener design used websocket `subscribeEvent`. A direct probe to `wss://fullnode.testnet.sui.io:443` returned HTTP `405`, so live mode was switched to HTTP `queryEvents` polling with backfill.
